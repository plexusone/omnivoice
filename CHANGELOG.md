## [v0.5.0] - 2026-03-01

### Highlights

- Batteries-included voice pipeline framework for Go with unified STT and TTS interfaces
- Provider registry pattern - consumers only need to import omnivoice, not individual provider packages
- Four providers registered: OpenAI, ElevenLabs, Deepgram, and Twilio

### Added

- Unified `STTClient` and `TTSClient` interfaces for speech-to-text and text-to-speech ([`66a341b`](https://github.com/plexusone/omnivoice/commit/66a341b))
- `TranscriptionConfig` and `SynthesisConfig` types for provider-agnostic configuration ([`66a341b`](https://github.com/plexusone/omnivoice/commit/66a341b))
- `providers/all` package for importing all providers with a single import ([`66a341b`](https://github.com/plexusone/omnivoice/commit/66a341b))
- Provider registry with `RegisterTTSProvider` and `RegisterSTTProvider` for self-registration
- `GetTTSProvider(name, opts...)` and `GetSTTProvider(name, opts...)` for runtime provider creation
- `WithAPIKey`, `WithBaseURL`, and `WithExtension` options for provider configuration
- `ListTTSProviders()` and `ListSTTProviders()` to discover registered providers
- Subtitle generation re-exported from omnivoice-core (`GenerateSRT`, `GenerateVTT`)
- Streaming support for real-time transcription and synthesis ([`66a341b`](https://github.com/plexusone/omnivoice/commit/66a341b))

### Documentation

- README with quick start guide, usage examples, and provider compatibility table ([`9835ed7`](https://github.com/plexusone/omnivoice/commit/9835ed7))
- Language codes reference table with BCP-47 examples
