## [v0.5.0] - 2026-03-01

### Highlights

- Batteries-included voice pipeline framework for Go with unified STT and TTS interfaces
- Provider registry pattern - consumers only need to import omnivoice, not individual provider packages
- Four providers registered: OpenAI, ElevenLabs, Deepgram, and Twilio

### Added

- Unified `STTProvider` and `TTSProvider` type aliases re-exported from omnivoice-core ([`545b928`](https://github.com/plexusone/omnivoice/commit/545b928))
- `TranscriptionConfig` and `SynthesisConfig` types for provider-agnostic configuration ([`545b928`](https://github.com/plexusone/omnivoice/commit/545b928))
- `providers/all` package for importing all providers with a single blank import ([`545b928`](https://github.com/plexusone/omnivoice/commit/545b928))
- Provider registry with `RegisterTTSProvider` and `RegisterSTTProvider` for self-registration ([`545b928`](https://github.com/plexusone/omnivoice/commit/545b928))
- `GetTTSProvider(name, opts...)` and `GetSTTProvider(name, opts...)` for runtime provider creation ([`545b928`](https://github.com/plexusone/omnivoice/commit/545b928))
- `WithAPIKey`, `WithBaseURL`, and `WithExtension` options for provider configuration ([`545b928`](https://github.com/plexusone/omnivoice/commit/545b928))
- `ListTTSProviders()` and `ListSTTProviders()` to discover registered providers ([`545b928`](https://github.com/plexusone/omnivoice/commit/545b928))
- `HasTTSProvider()` and `HasSTTProvider()` to check provider availability ([`545b928`](https://github.com/plexusone/omnivoice/commit/545b928))
- Subtitle generation re-exported from omnivoice-core (`GenerateSRT`, `GenerateVTT`) ([`545b928`](https://github.com/plexusone/omnivoice/commit/545b928))
